  <action>
    Modify the `submitAssessmentAnswers` (or equivalent `/submit`) endpoint in AssessmentAnswerController:

    1. Add `@Autowired AssessmentSessionService assessmentSessionService` field

    2. At the START of the submit method, after extracting `userStudentId` and `assessmentId` from the request body, add the idempotency check:
       ```java
       // Idempotency check — SET NX prevents duplicate processing
       if (!assessmentSessionService.acquireSubmissionLock(userStudentId, assessmentId)) {
           Object cachedResult = assessmentSessionService.getSubmissionResult(userStudentId, assessmentId);
           if (cachedResult != null && !"processing".equals(cachedResult)) {
               // Return cached successful result
               return ResponseEntity.ok(cachedResult);
           }
           // Still processing (concurrent request) or no result yet
           return ResponseEntity.status(HttpStatus.CONFLICT).body(Map.of(
               "error", "Submission already in progress or completed",
               "status", "duplicate"
           ));
       }
       ```

    3. Wrap the EXISTING submission logic (everything after the idempotency check) in a try-catch:
       ```java
       try {
           // ... existing submission logic (validate entities, delete old answers, save new answers, save raw scores) ...

           // Build result map (use the existing response structure)
           Map<String, Object> result = new HashMap<>();
           result.put("status", "success");
           // ... add existing response fields ...

           // Cache result and clean up session
           assessmentSessionService.markSubmissionComplete(userStudentId, assessmentId, result);
           assessmentSessionService.deleteSession(userStudentId, assessmentId);

           return ResponseEntity.ok(result);
       } catch (Exception e) {
           // Release idempotency lock on failure so student can retry
           assessmentSessionService.clearSubmissionLock(userStudentId, assessmentId);
           throw e; // Let existing error handling catch it
       }
       ```

    4. Do NOT change the existing submission logic (delete old answers, save new answers, save raw scores). Only add the idempotency wrapper around it.

    5. Do NOT add idempotency to bulk-submit endpoints (`/bulk-submit`, `/bulk-submit-with-students`, `/bulk-submit-by-rollnumber`). These are admin-only and handle re-upload by deleting old answers first.

    6. Add import for `com.kccitm.api.service.AssessmentSessionService` and `org.springframework.http.HttpStatus`.
