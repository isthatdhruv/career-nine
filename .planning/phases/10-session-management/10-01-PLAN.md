    - `getSubmissionResult(Long studentId, Long assessmentId)` — gets value from submit key.
    - `clearSubmissionLock(Long studentId, Long assessmentId)` — deletes the submit key.
    - Use SLF4J logger for info-level logging on session create/validate/delete.

        if (value instanceof java.util.Map) {

            return mapper.convertValue(value, AssessmentSession.class);
        }
        return null;
    }

    Create `AssessmentSessionInterceptor.java` in `com.kccitm.api.config` package:
    - Implements `HandlerInterceptor`
    - Annotated `@Component`
    - Inject `AssessmentSessionService`
    - In `preHandle()`:
      1. Read `X-Assessment-Session` header from request
      2. If header is null or empty, return true (backwards compatible — no header = pass through)
      3. Read `X-Assessment-Student-Id` and `X-Assessment-Id` headers (frontend will send these alongside the session token — avoids reading request body which is consumed once)
      4. If student ID or assessment ID headers are missing, return true (can't validate without IDs)
      5. Parse both IDs as Long
      6. Call `sessionService.validateSession(studentId, assessmentId, sessionToken)`
      7. If null (invalid/expired): set response status 403, write JSON `{"error":"Invalid or expired assessment session"}`, return false
      8. If valid: set request attribute `assessmentSession` with the session, return true
    - Log WARN on invalid session attempts (studentId, assessmentId, but not the token)

    Modify `WebMvcConfig.java`:
    - Add `@Autowired AssessmentSessionInterceptor sessionInterceptor`
    - Override `addInterceptors(InterceptorRegistry registry)`:
      ```java
      registry.addInterceptor(sessionInterceptor)
          .addPathPatterns("/assessment-answer/**", "/assessments/**")
          .excludePathPatterns(
              "/assessments/getAll",
              "/assessments/get/list*",
              "/assessments/create",
              "/assessments/update/**",
              "/assessments/delete/**",
              "/assessments/prefetch/**",
              "/assessments/startAssessment"
          );
      ```
    - Note: startAssessment is EXCLUDED because it creates the session (can't validate what doesn't exist yet)
    - Keep existing CORS and multipart config unchanged

    - Do NOT change the method signature, return type, or existing logic
