<task type="auto">
  <name>Task 1: Replace CaffeineCacheManager with RedisCacheManager in CacheConfig.java</name>
  <files>spring-social/src/main/java/com/kccitm/api/config/CacheConfig.java</files>
  <action>
Rewrite CacheConfig.java to use RedisCacheManager instead of CaffeineCacheManager. Keep the @Primary annotation so existing @Cacheable annotations pick it up.

**Implementation details:**

1. Remove all Caffeine imports (CaffeineCacheManager, Caffeine, TimeUnit).
2. Add imports for: `org.springframework.data.redis.cache.RedisCacheConfiguration`, `org.springframework.data.redis.cache.RedisCacheManager`, `org.springframework.data.redis.connection.RedisConnectionFactory`, `org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer`, `org.springframework.data.redis.serializer.RedisSerializationContext`, `org.springframework.data.redis.serializer.StringRedisSerializer`, `java.time.Duration`, `java.util.HashMap`, `java.util.Map`.
3. Inject `RedisConnectionFactory` via constructor or method parameter.
4. Create a `RedisCacheConfiguration` default config with:
   - TTL of 1 day (`Duration.ofDays(1)`) — matches current Caffeine expireAfterWrite
   - Key serializer: `StringRedisSerializer` — human-readable keys
   - Value serializer: `GenericJackson2JsonRedisSerializer` — matches RedisConfig pattern from Phase 8
   - Key prefix: `"career9:"` — namespace all keys to avoid collisions
   - Disable caching null values: `disableCachingNullValues()`
5. Build `RedisCacheManager` with `RedisCacheManager.builder(connectionFactory)`:
   - Set `cacheDefaults(defaultConfig)`
   - Use `withInitialCacheConfigurations(Map)` to explicitly register the 3 cache names: "assessmentDetails", "assessmentQuestions", "measuredQualityTypes" — all using default config
   - Call `.transactionAware()` so cache operations participate in Spring transactions
6. Keep the bean name as `cacheManager()` and keep `@Primary`.
7. Remove the Caffeine dependency usage but do NOT remove caffeine from pom.xml (it may be used elsewhere).

**Why GenericJackson2JsonRedisSerializer:** Same serializer used by RedisTemplate (Phase 8 decision). Enables human-readable Redis values for debugging. The serializer stores type info (@class) in JSON so deserialization works without knowing the target type at compile time.

**Circular reference concern (from STATE.md blocker):** AssessmentQuestions has bidirectional relationships (question -> options -> scores -> question_option back-ref). GenericJackson2JsonRedisSerializer uses Jackson which will follow @JsonManagedReference/@JsonBackReference if present. If not present and circular refs exist, Jackson will throw. The existing Caffeine cache already caches these objects successfully via the same controller responses (which go through Jackson serialization for HTTP). Therefore Redis serialization should work identically — the HTTP response serialization is proof the object graph is Jackson-safe.

<task type="auto">
  <name>Task 2: Switch spring.cache.type to redis in application.yml for all profiles</name>
  <files>spring-social/src/main/resources/application.yml</files>
  <action>
Update application.yml to switch the cache type from caffeine to redis across all three profiles (dev, staging, production).

**For each profile section (dev, staging, production):**

1. Change `spring.cache.type: caffeine` to `spring.cache.type: redis`
2. Remove the `caffeine:` sub-section with `spec: maximumSize=500,expireAfterWrite=600s` — these are Caffeine-specific settings no longer needed since RedisCacheManager handles TTL and size via Redis maxmemory policy.
3. Keep the `cache-names:` list as-is — Spring uses it for auto-creating caches. The list should remain: `assessmentQuestions,assessmentDetails,measuredQualityTypes,questionnaireQuestions`
4. Add `spring.cache.redis.time-to-live: 86400000` (1 day in milliseconds) as a fallback TTL. The RedisCacheManager in CacheConfig.java sets TTL programmatically, but this property ensures Spring Boot auto-configuration also uses the right TTL if the bean isn't loaded for some reason.
5. Add `spring.cache.redis.key-prefix: "career9:"` as documentation/fallback.
6. Add `spring.cache.redis.use-key-prefix: true` to ensure keys are namespaced.

**Do NOT change:** spring.redis.* connection properties (host, port, pool settings) — those are from Phase 8 and must remain unchanged.
  </action>
